# Contributing to AUSTSat Mission Control Dashboard

First off, thank you for considering contributing to the AUSTSat Mission Control project! Every contribution, from a small typo fix to a new feature, is valuable. This document provides a set of guidelines to help you get started.

## How Can I Contribute?

There are many ways to contribute, and we welcome all of them:

-   **Reporting Bugs:** Find a problem? Let us know!
-   **Suggesting Enhancements:** Have an idea to make the dashboard better?
-   **Improving Documentation:** See a typo or a confusing section? Fix it!
-   **Writing Code:** Want to get your hands dirty and fix a bug or add a new feature?

## Getting Started

Before you start, please check the [Issues tab](https://github.com/abrar-khan-alvi/AUSTSat_Frontend/issues) to see if someone else has already reported your bug or suggested your idea. This helps us avoid duplicate efforts.

-   Look for issues tagged with `good first issue` if you are new to the project. These are designed to be a great way to get your feet wet.
-   If you plan to work on an existing issue, please leave a comment so we can assign it to you and avoid having multiple people work on the same thing.

### Reporting Bugs

-   Go to the [Issues tab](https://github.com/abrar-khan-alvi/AUSTSat_Frontend/issues) and click "New Issue".
-   Use the "Bug Report" template if available.
-   Provide a clear and descriptive title.
-   Describe the exact steps to reproduce the bug.
-   Include screenshots or animated GIFs if they help explain the problem.

### Suggesting Enhancements

-   Go to the [Issues tab](https://github.com/abrar-khan-alvi/AUSTSat_Frontend/issues) and click "New Issue".
-   Use the "Feature Request" template if available.
-   Clearly describe the proposed enhancement and why it would be useful.

## Submitting a Pull Request

Ready to contribute some code? Great! Here is the process:

1.  **Fork the Repository**
    -   Click the "Fork" button at the top right of the repository page. This creates your own copy of the project.

2.  **Clone Your Fork**
    -   Clone your forked repository to your local machine.
    ```bash
    git clone https://github.com/YOUR_USERNAME/AUSTSat_Frontend.git
    cd AUSTSat_Frontend
    ```

3.  **Create a New Branch**
    -   Create a descriptive branch name for your changes.
    ```bash
    git checkout -b feature/add-new-chart
    # or for a bug fix:
    # git checkout -b fix/dashboard-layout-issue
    ```

4.  **Make Your Changes**
    -   Write your code! Make sure to follow the local setup instructions in the `README.md` to run the project.

5.  **Commit Your Changes**
    -   Commit your changes with a clear and concise commit message.
    ```bash
    git add .
    git commit -m "feat: Add atmospheric pressure chart to analysis page"
    ```
    -   We follow the [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) specification. Common prefixes include `feat:`, `fix:`, `docs:`, `style:`, `refactor:`.

6.  **Push to Your Fork**
    -   Push your new branch to your forked repository on GitHub.
    ```bash
    git push origin feature/add-new-chart
    ```

7.  **Open a Pull Request**
    -   Go to your fork on GitHub. You will see a "Compare & pull request" button. Click it.
    -   Provide a clear title and a detailed description of the changes you've made. Link to any relevant issues (e.g., "Closes #12").
    -   Submit the pull request. We will review it as soon as possible!

Thank you for your contribution!
